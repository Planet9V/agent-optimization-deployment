## 1. Theoretical Foundation

### 1.1 McKenney's 10 Strategic Questions

**Source:** McKenney, J. (2023-2025). *Strategic Questioning Framework for Cyber Threat Intelligence*

McKenney's framework organizes cyber threat analysis around 10 fundamental questions that guide defenders from detection to improvement:

1. **Q1: Who threatens us?** → Threat actor identification
2. **Q2: What do they want?** → Motivation analysis
3. **Q3: What are they doing?** → TTP (Tactics, Techniques, Procedures) observation
4. **Q4: Where are they attacking?** → Asset targeting analysis
5. **Q5: How are they attacking?** → Technical methodology
6. **Q6: What's the impact?** → Damage assessment
7. **Q7: What threats are coming?** → Predictive intelligence
8. **Q8: What to patch first?** → Prioritization
9. **Q9: How to defend?** → Control implementation
10. **Q10: How to improve?** → Feedback and learning

**Traditional Limitation:** McKenney's questions address *observable behaviors* but lack a theory of **unconscious motivation**, **organizational defense mechanisms**, and **cultural symbolic orders** that determine why threats succeed or fail.

### 1.2 Lacanian Psychoanalytic Theory

**Source:** Lacan, J. (1966/2006). *Écrits*. W.W. Norton. ISBN:9780393061154

Jacques Lacan's psychoanalytic framework posits that human experience operates across **three registers**:

#### The Real (R)
- **Definition:** Unmediated, traumatic experience that resists symbolization
- **Characteristics:** Anxiety-provoking, impossible to fully grasp, returns as symptom
- **Cyber Application:** Actual breach impact, unprocessed threat intelligence, zero-day chaos

**Example:** A healthcare organization experiences a ransomware attack. The Real is:
- Physical inability to access patient records (traumatic, immediate)
- Unprocessed fear that "we don't know what else they did"
- The gap between security controls (Symbolic) and actual breach (Real)

#### The Imaginary (I)
- **Definition:** Realm of ego, self-image, and mirror stage identification
- **Characteristics:** Narcissistic investment, rivalry, gestalt perception
- **Cyber Application:** Threat actor brand/reputation, organizational security theater, perceived invulnerability

**Example:** A nation-state APT group:
- Cultivates "sophisticated" image (Imaginary self-identification)
- Engages in symbolic gestures (defacement with political messages)
- Derives jouissance from media coverage, not just intelligence gain

#### The Symbolic (S)
- **Definition:** Language, law, social norms, the "big Other"
- **Characteristics:** Structured like a language, determines subject position, enables communication
- **Cyber Application:** Cybersecurity policies, compliance frameworks (NIST, ISO 27001), cultural security norms

**Example:** GDPR compliance:
- Symbolic law inscribing subject position ("data controller", "data subject")
- Symbolic punishment (fines) maintaining social order
- Tension with Real (actual breach chaos) and Imaginary (organizational self-image as "secure")

### 1.3 Statistical Physics: Ising Model for Cultural Dynamics

**Source:** Castellano, C., et al. (2009). Statistical physics of social dynamics. *Reviews of Modern Physics*, 81(2), 591. doi:10.1103/RevModPhys.81.591

The **Ising model** from statistical physics models how individual "spins" (binary opinions) align or misalign under social influence. Applied to cybersecurity culture:

**Ising Hamiltonian:**
```
H = -J Σ sᵢ sⱼ - h Σ sᵢ
    <i,j>        i

sᵢ = opinion of individual i (+1 or -1)
J = coupling strength (social influence)
h = external field (training, propaganda)
<i,j> = sum over neighboring pairs
```

**Magnetization (Cultural Coherence):**
```
m = (1/N) Σ sᵢ
         i

m ≈ +1: Strong cultural consensus (e.g., "security is everyone's job")
m ≈ 0: No consensus, cultural chaos
m ≈ -1: Strong consensus in opposition (e.g., "security theater is pointless")
```

**Dynamics (Opinion Evolution):**
```
dm/dt = -m + tanh(β(Jzm + h))

β = inverse temperature (cultural rigidity)
z = average number of connections
```

**Critical Insight:** Lacanian Symbolic register (policies, norms) maps directly to Ising dynamics. Cultural coherence predicts policy adoption success.

**Traditional McKenney Approach:**

**Lacanian Enhancement:** Patching = Symbolic repair; prioritization must account for organizational capacity (Real) and morale (Imaginary)

**Psychohistory Model:** Epidemic R₀ (vulnerability spread) × Workforce capacity

**Cypher Query:**
```cypher

// Real: Technical severity and asset criticality
WITH vuln, asset,
     vuln.cvss_score AS real_severity,
     asset.criticality_score AS real_asset_value

// Symbolic: Exploit availability (symbolic transgression likelihood)
OPTIONAL MATCH (vuln)<-[:EXPLOITS]-(exploit:AttackPattern)
WITH vuln, asset, real_severity, real_asset_value,
     count(exploit) AS exploit_count,
     CASE WHEN count(exploit) > 0 THEN 1.5 ELSE 1.0 END AS symbolic_urgency_multiplier

// Epidemic spread potential (R₀ calculation)
OPTIONAL MATCH (vuln)-[:SPREADS_VIA]->(protocol:Standard)
WITH vuln, asset, real_severity, real_asset_value, symbolic_urgency_multiplier,
     CASE
       WHEN protocol.name CONTAINS 'SMB' OR protocol.name CONTAINS 'RDP' THEN 2.5
       WHEN protocol.name CONTAINS 'HTTP' THEN 1.8
       ELSE 1.0
     END AS R0_multiplier

// Imaginary: Workforce capacity (burnout reduces patching effectiveness)
MATCH (org:Organization)-[:OWNS]->(asset)
OPTIONAL MATCH (org)-[:EMPLOYS]->(security_team:Role {department: 'Security'})
OPTIONAL MATCH (security_team)-[:EXHIBITS_TRAIT]->(burnout:PsychTrait {metric_name: 'Burnout_Score'})
WITH vuln, asset, real_severity, real_asset_value, symbolic_urgency_multiplier, R0_multiplier,
     avg(burnout.current_value) AS avg_team_burnout,
     count(security_team) AS team_size

WITH vuln, asset,
     // McKenney-Lacan Priority Score
     real_severity * real_asset_value * symbolic_urgency_multiplier * R0_multiplier *
     // Workforce capacity adjustment (burnout reduces effectiveness)
     (1.0 - avg_team_burnout * 0.5) *
     // Team size factor
     (1.0 / sqrt(team_size)) AS mckenney_lacan_priority_score

RETURN vuln.cve_id,
       asset.name,
       mckenney_lacan_priority_score,
       CASE
         WHEN mckenney_lacan_priority_score > 50.0 THEN 'PRIORITY 1 (Patch immediately)'
         WHEN mckenney_lacan_priority_score > 20.0 THEN 'PRIORITY 2 (Patch within 7 days)'
         WHEN mckenney_lacan_priority_score > 10.0 THEN 'PRIORITY 3 (Patch within 30 days)'
         ELSE 'PRIORITY 4 (Standard cycle)'
       END AS patching_priority
ORDER BY mckenney_lacan_priority_score DESC
LIMIT 20;
```

### 4.5 Granovetter Cascade: Mapping All Registers to Social Tipping Points

**Threshold Cascade Model:**
```
r(t+1) = N × F(r(t)/N)

r(t) = number of adopters at time t
N = population size
F = CDF of threshold distribution
F(p) = ∫₀ᵖ f(θ) dθ  (fraction with threshold ≤ p)
```

**Cybersecurity Adaptation:**

| Cascade Variable | Cyber Interpretation |
|------------------|----------------------|
| r(t) | Number adopting security practice |
| N | Total population (employees, threat actors, etc.) |
| F | Threshold distribution (individual adoption thresholds) |
| f(θ) | Threshold density (typically normal: N(μ, σ²)) |

**Lacanian Mapping:**
- **Real:** Actual adoption count r(t)
- **Imaginary:** Individual threshold θᵢ (ego investment in being "early adopter" vs. "follower")
- **Symbolic:** Distribution F shaped by cultural norms (low μ = early adoption culture)

**Example: MFA Adoption Cascade**

```cypher

     toFloat(current_adopters) / N AS p_current,
     // Threshold distribution parameters (normal)

     0.15 AS sigma_threshold  // Std dev

WITH org, N, current_adopters, p_current, mu_threshold, sigma_threshold,
     // Granovetter cascade prediction
     // F(p) ≈ Φ((p - μ) / σ)  for normal distribution
     // New adopters = N × Φ((p_current - μ) / σ)
     N * (0.5 + 0.5 * erf((p_current - mu_threshold) / (sigma_threshold * sqrt(2.0)))) AS predicted_next_period

       N AS total_employees,
       current_adopters,
       p_current AS current_adoption_fraction,
       toInteger(predicted_next_period) AS forecast_adopters_next_quarter,
       toInteger(predicted_next_period - current_adopters) AS incremental_adoption,
       CASE
         WHEN p_current > mu_threshold + sigma_threshold THEN
           'CASCADE ACCELERATING (past tipping point)'
         WHEN p_current > mu_threshold THEN
           'CASCADE INITIATED (past average threshold)'
         ELSE
           'PRE-CASCADE (early adopters only)'
       END AS cascade_stage;
```

**Tipping Point:** μ = 0.4 means cascade accelerates once 40% adopt (critical mass)
