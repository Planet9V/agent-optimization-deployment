# Enneagram Type 2: The Helper/Giver

## Core Personality Traits

**PERSONALITY_TRAIT**: Compulsive need to be helpful and indispensable to colleagues, offering unsolicited security assistance
**INSIDER_INDICATOR**: May create security dependencies or problems to solve, ensuring continued relevance
**THREAT_ACTOR**: Helper-turned-saboteur who creates incidents requiring their expertise

**PERSONALITY_TRAIT**: Seeks appreciation and recognition through security contributions, becomes resentful when unacknowledged
**INSIDER_INDICATOR**: High sabotage risk if feels taken for granted - may withhold security knowledge or assistance
**COGNITIVE_BIAS**: Reciprocity expectation - assumes help given entitles them to help received

**PERSONALITY_TRAIT**: Manipulative helpfulness - assists others strategically to build influence and gain access
**INSIDER_INDICATOR**: Uses security assistance to gather information or create obligations for later exploitation
**THREAT_ACTOR**: Social engineering insider who builds trust through helpful behavior before exploitation

**PERSONALITY_TRAIT**: Pride in being security expert others depend on, enjoys being irreplaceable
**INSIDER_INDICATOR**: May intentionally create knowledge silos or undocumented security configurations
**COGNITIVE_BIAS**: Illusion of uniqueness - overestimates how much organization depends on their specific expertise

**PERSONALITY_TRAIT**: Difficulty setting boundaries with security requests, saying yes to maintain relationships
**INSIDER_INDICATOR**: Grants excessive access or permissions to maintain positive relationships
**COGNITIVE_BIAS**: Social proof bias - approves security requests if others approved similar requests

**PERSONALITY_TRAIT**: Represses own security needs and concerns while focusing on others' security challenges
**INSIDER_INDICATOR**: May neglect personal security hygiene or fail to report own security incidents
**COGNITIVE_BIAS**: Spotlight effect (inverted) - underestimates how much others notice personal security lapses

**PERSONALITY_TRAIT**: Flattery and charm used to build relationships with security stakeholders
**INSIDER_INDICATOR**: Positive trait when authentic - builds trust necessary for security collaboration
**COGNITIVE_BIAS**: Liking bias - people grant security access to those they like

**PERSONALITY_TRAIT**: Possessive about security relationships, becomes jealous when others provide security assistance
**INSIDER_INDICATOR**: May undermine other security personnel to maintain unique helper role
**THREAT_ACTOR**: Territorial insider who sabotages competing security initiatives

**PERSONALITY_TRAIT**: Strong emotional intelligence and ability to read security stakeholders' unspoken needs
**INSIDER_INDICATOR**: Positive trait - anticipates security concerns and addresses them proactively
**COGNITIVE_BIAS**: Empathy bias - may grant exceptions based on understanding emotional context

**PERSONALITY_TRAIT**: Gives security help with strings attached, though often unspoken expectations
**INSIDER_INDICATOR**: May demand reciprocal favors or use past security assistance as leverage
**COGNITIVE_BIAS**: Reciprocity bias - expects equivalent return on security assistance provided

## Wing Variations

### Type 2w1 (Wing One - The Servant)

**PERSONALITY_TRAIT**: Combines helpfulness with principled approach to security assistance
**INSIDER_INDICATOR**: More ethical and less manipulative than core Type 2, but still seeks appreciation
**COGNITIVE_BIAS**: Moral licensing - feels entitled to recognition after ethical security service

**PERSONALITY_TRAIT**: Self-critical about quality of security help provided, striving for perfect assistance
**INSIDER_INDICATOR**: May overwork to maintain both helpful and excellent standards
**COGNITIVE_BIAS**: Perfectionism bias - rejects adequate security solutions in favor of ideal help

**PERSONALITY_TRAIT**: More reserved and formal when offering security assistance, less emotionally expressive
**INSIDER_INDICATOR**: Lower manipulation risk - helps from genuine principle rather than emotional need
**COGNITIVE_BIAS**: Authority bias - defers to official security policies while helping others

**PERSONALITY_TRAIT**: Becomes rigid and judgmental when security help is rejected or unappreciated
**INSIDER_INDICATOR**: May withdraw assistance entirely if feels moral standards aren't appreciated
**THREAT_ACTOR**: Principled helper who justifies sabotage when organization violates ethical standards

### Type 2w3 (Wing Three - The Host/Hostess)

**PERSONALITY_TRAIT**: Combines helpfulness with image-consciousness and ambition for security recognition
**INSIDER_INDICATOR**: Higher manipulation risk - strategic helpfulness to build career and reputation
**COGNITIVE_BIAS**: Self-serving bias - attributes security success to personal help, failures to others

**PERSONALITY_TRAIT**: Charismatic and socially skilled in building security relationships across organization
**INSIDER_INDICATOR**: Positive trait - creates security culture through relationship building
**COGNITIVE_BIAS**: Halo effect - charm creates positive bias in security decisions

**PERSONALITY_TRAIT**: Competitive about being most helpful security resource, jealous of rival helpers
**INSIDER_INDICATOR**: May sabotage competing security initiatives or undermine other security personnel
**THREAT_ACTOR**: Competitive insider who eliminates security rivals through sabotage

**PERSONALITY_TRAIT**: Adapts helping style to audience, providing different security assistance based on stakeholder
**INSIDER_INDICATOR**: Positive trait - tailors security communication effectively; Risk - inconsistent security guidance
**COGNITIVE_BIAS**: Framing effect - presents security recommendations differently based on audience

## Integration and Disintegration

### Movement to Type 4 (Integration/Growth)

**PERSONALITY_TRAIT**: Becomes more authentic about own security needs rather than exclusively focusing on others
**INSIDER_INDICATOR**: Reduced manipulation risk - asks directly for security resources rather than expecting reciprocity
**COGNITIVE_BIAS**: Self-awareness - recognizes own biases and security blind spots

**PERSONALITY_TRAIT**: Develops emotional depth and accepts that being helpful isn't only source of worth
**INSIDER_INDICATOR**: Positive development - more balanced security relationships without dependency creation
**COGNITIVE_BIAS**: Introspection bias - better understanding of own security motivations

**PERSONALITY_TRAIT**: Sets healthy boundaries on security assistance, saying no without guilt
**INSIDER_INDICATOR**: Reduced burnout risk - sustainable security workload and relationships
**COGNITIVE_BIAS**: Loss aversion - becomes comfortable with others' disappointment in limited help

### Movement to Type 8 (Disintegration/Stress)

**PERSONALITY_TRAIT**: Becomes aggressive and demanding when security help isn't appreciated
**INSIDER_INDICATOR**: High sabotage risk - resentful manipulation turns to overt aggression
**THREAT_ACTOR**: Resentful helper who sabotages those perceived as ungrateful

**PERSONALITY_TRAIT**: Openly expresses anger about being taken for granted in security role
**INSIDER_INDICATOR**: Warning sign - shift from passive to active aggression regarding recognition
**COGNITIVE_BIAS**: Fundamental attribution error - blames others' character for not appreciating help

**PERSONALITY_TRAIT**: Becomes controlling and domineering in security decisions previously offered as help
**INSIDER_INDICATOR**: May abuse security privileges to punish those who didn't reciprocate assistance
**THREAT_ACTOR**: Control-seeking insider who enforces security through intimidation

**PERSONALITY_TRAIT**: Asserts own security needs aggressively after long period of repression
**INSIDER_INDICATOR**: Dramatic shift from selfless to selfish security behavior signals stress
**COGNITIVE_BIAS**: Restraint bias - overestimated ability to continue suppressing own needs

## Subtype Variations

### Self-Preservation Two (Privilege Subtype)

**PERSONALITY_TRAIT**: Uses security helpfulness to create comfort and privilege for self
**INSIDER_INDICATOR**: Grants security assistance expecting tangible returns - resources, status, perks
**COGNITIVE_BIAS**: Reciprocity bias - explicit expectation of material reciprocation

**PERSONALITY_TRAIT**: More childlike and innocent presentation when requesting security access or resources
**INSIDER_INDICATOR**: May use seeming naivety to gain inappropriate security permissions
**THREAT_ACTOR**: Innocent-appearing insider who exploits trust built through helpfulness

**PERSONALITY_TRAIT**: Focuses security assistance on those who can provide material benefits
**INSIDER_INDICATOR**: Strategic rather than universal helpfulness - targets influential stakeholders
**COGNITIVE_BIAS**: Availability heuristic - helps those most visible and likely to reciprocate

**PERSONALITY_TRAIT**: Fears being without security resources or support, hoards security knowledge
**INSIDER_INDICATOR**: May withhold security documentation to maintain indispensable status
**COGNITIVE_BIAS**: Scarcity mindset - assumes security knowledge shared is security value lost

### Social Two (Ambition Subtype)

**PERSONALITY_TRAIT**: Uses security helpfulness to build influential network across organization
**INSIDER_INDICATOR**: Positive trait - creates security culture; Risk - uses security access for political positioning
**COGNITIVE_BIAS**: Halo effect - security expertise creates positive bias in non-security contexts

**PERSONALITY_TRAIT**: Desires to be important and powerful through security role and relationships
**INSIDER_INDICATOR**: May exaggerate security threats to increase own importance and influence
**THREAT_ACTOR**: Self-important insider who creates security drama for attention

**PERSONALITY_TRAIT**: Adapts security persona to fit with desired social group or leadership tier
**INSIDER_INDICATOR**: May compromise security standards to fit with executive culture
**COGNITIVE_BIAS**: Social proof bias - aligns security recommendations with leadership preferences

**PERSONALITY_TRAIT**: Competes for status as most helpful security resource in organization
**INSIDER_INDICATOR**: May undermine other security personnel to maintain unique position
**THREAT_ACTOR**: Status-seeking insider who sabotages competing security initiatives

### Sexual Two (Seduction/Aggression Subtype)

**PERSONALITY_TRAIT**: Uses charm and seduction to build intense security relationships with key stakeholders
**INSIDER_INDICATOR**: May develop inappropriate personal relationships leveraging security role
**THREAT_ACTOR**: Seductive insider who uses romantic/sexual relationships to gain security access

**PERSONALITY_TRAIT**: Wants to be desired and needed intensely by security stakeholders
**INSIDER_INDICATOR**: Creates security crises or dependencies to be indispensable
**COGNITIVE_BIAS**: Confirmation bias - interprets interactions as proof of being needed

**PERSONALITY_TRAIT**: Aggressive pursuit of security relationships with influential targets
**INSIDER_INDICATOR**: Persistent boundary violations disguised as helpful security attention
**THREAT_ACTOR**: Stalker-type insider who won't accept rejection of security assistance

**PERSONALITY_TRAIT**: Experiences jealousy when security stakeholders rely on others for assistance
**INSIDER_INDICATOR**: May sabotage competing security relationships or undermine rivals
**THREAT_ACTOR**: Jealous insider who eliminates perceived security competitors

## Security Role Suitability

**PERSONALITY_TRAIT**: Excellent in security awareness and training roles requiring relationship building
**INSIDER_INDICATOR**: Positive trait - natural ability to make security accessible and build culture
**COGNITIVE_BIAS**: Liking bias - people adopt security practices from trainers they like

**PERSONALITY_TRAIT**: Strong performance in customer-facing security support and helpdesk roles
**INSIDER_INDICATOR**: Positive trait - patient and empathetic security assistance
**COGNITIVE_BIAS**: Empathy bias - may grant exceptions based on understanding user frustrations

**PERSONALITY_TRAIT**: Challenges in security enforcement roles requiring saying no and setting boundaries
**INSIDER_INDICATOR**: May grant excessive permissions or overlook violations to maintain relationships
**COGNITIVE_BIAS**: Social proof bias - approves requests similar to those previously approved

**PERSONALITY_TRAIT**: Risk in access control roles due to difficulty refusing security requests
**INSIDER_INDICATOR**: Inappropriate access grants to maintain positive relationships
**THREAT_ACTOR**: Permissive insider who grants access to maintain likability

**PERSONALITY_TRAIT**: Creates security dependencies through undocumented or complex configurations only they understand
**INSIDER_INDICATOR**: Knowledge hoarding to ensure continued relevance and appreciation
**COGNITIVE_BIAS**: Illusion of explanatory depth - believes only they can understand their security implementations

## Team Dynamics and Insider Risk

**PERSONALITY_TRAIT**: Builds strong security culture through relationship-based approach and helpfulness
**INSIDER_INDICATOR**: Positive trait - team trusts and collaborates on security initiatives
**COGNITIVE_BIAS**: In-group bias - preferentially helps team members over outsiders

**PERSONALITY_TRAIT**: Creates unhealthy team dependencies where others defer security decisions to the helper
**INSIDER_INDICATOR**: Team lacks security self-sufficiency, vulnerable if helper leaves or becomes malicious
**COGNITIVE_BIAS**: Authority bias - team members defer to helper's security judgment without questioning

**PERSONALITY_TRAIT**: Becomes resentful when team members don't express appreciation for security assistance
**INSIDER_INDICATOR**: Warning sign - declining helpfulness or passive-aggressive withholding of security support
**COGNITIVE_BIAS**: Negativity bias - focuses on instances of ingratitude rather than appreciation received

**PERSONALITY_TRAIT**: Manipulates team dynamics by selectively helping members who show appreciation
**INSIDER_INDICATOR**: Creates security information inequality and team division
**THREAT_ACTOR**: Divisive insider who uses security knowledge as social currency

**PERSONALITY_TRAIT**: Struggles when new security team members provide assistance, feeling territorial
**INSIDER_INDICATOR**: May undermine new security personnel to maintain unique helper status
**THREAT_ACTOR**: Territorial insider who sabotages competing security expertise

## Stress Responses and Warning Signs

**PERSONALITY_TRAIT**: Under stress, moves to Type 8 - helpfulness becomes aggressive control
**INSIDER_INDICATOR**: Critical warning sign - shift from manipulation to overt abuse of security access
**THREAT_ACTOR**: Stressed helper who weaponizes security access against perceived ingrates

**PERSONALITY_TRAIT**: Expresses anger about feeling taken for granted after years of security assistance
**INSIDER_INDICATOR**: High-risk warning sign - resentment accumulation precedes sabotage
**COGNITIVE_BIAS**: Anchoring bias - fixates on negative interactions while discounting positive appreciation

**PERSONALITY_TRAIT**: Withdraws security assistance entirely or provides only minimal help
**INSIDER_INDICATOR**: Passive-aggressive response to perceived lack of appreciation
**THREAT_ACTOR**: Withholding insider who creates security gaps through withdrawn support

**PERSONALITY_TRAIT**: Begins demanding explicit recognition or compensation for security contributions
**INSIDER_INDICATOR**: Warning sign - shift from implicit to explicit reciprocity expectations
**COGNITIVE_BIAS**: Endowment effect - overvalues own security contributions

## Mitigation Strategies

**PERSONALITY_TRAIT**: Requires regular recognition and appreciation for security contributions
**INSIDER_INDICATOR**: Reduces sabotage risk by meeting need for acknowledgment
**COGNITIVE_BIAS**: Recency bias - recent recognition has outsized positive impact

**PERSONALITY_TRAIT**: Benefits from clear security role definitions that include and value helping behaviors
**INSIDER_INDICATOR**: Channels helpfulness productively when formally recognized as valuable
**COGNITIVE_BIAS**: Confirmation bias - seeks evidence that helpfulness is officially valued

**PERSONALITY_TRAIT**: Needs training on healthy boundaries and appropriate security assistance limits
**INSIDER_INDICATOR**: Reduces inappropriate access grants and security exceptions
**COGNITIVE_BIAS**: Authority bias - respects boundary guidance from security leadership

**PERSONALITY_TRAIT**: Requires documentation requirements to prevent knowledge hoarding
**INSIDER_INDICATOR**: Reduces dependency risk and ensures security continuity
**COGNITIVE_BIAS**: Loss aversion - documentation requirements overcome reluctance to share knowledge

**PERSONALITY_TRAIT**: Benefits from peer recognition programs where team expresses appreciation
**INSIDER_INDICATOR**: Meets need for acknowledgment in healthy, sustainable way
**COGNITIVE_BIAS**: Social proof - peer appreciation has high value for social Type 2

**INSIDER_INDICATOR**: Strong emotional reactions to perceived ingratitude increase sabotage risk
**COGNITIVE_BIAS**: Availability heuristic - emotional incidents are more memorable and influential

**INSIDER_INDICATOR**: High risk - combines manipulation with security access for exploitation

**INSIDER_INDICATOR**: May compromise security for relationship maintenance
**COGNITIVE_BIAS**: Affect heuristic - security decisions influenced by emotional relationships

**PERSONALITY_TRAIT**: Anxious attachment (Attachment Theory) - fears abandonment if not helpful enough
**INSIDER_INDICATOR**: Excessive helpfulness and difficulty setting security boundaries
**COGNITIVE_BIAS**: Catastrophizing - assumes minor security refusals will destroy relationships

**INSIDER_INDICATOR**: Creates security situations requiring their intervention for recognition
**THREAT_ACTOR**: Attention-seeking insider who manufactures security crises

**INSIDER_INDICATOR**: Positive trait - authentic helpfulness rather than manipulative exploitation
**COGNITIVE_BIAS**: Empathy bias - security decisions influenced by genuine concern for users

## Assessment and Detection

**PERSONALITY_TRAIT**: Look for excessive eagerness to help during security onboarding and initial interactions
**INSIDER_INDICATOR**: Boundary-crossing helpfulness may indicate Type 2 manipulation patterns
**COGNITIVE_BIAS**: Halo effect - initial positive impression biases ongoing assessment

**PERSONALITY_TRAIT**: Monitor for security assistance with implicit strings attached or expectations
**INSIDER_INDICATOR**: Warning sign - helpfulness designed to create obligations
**COGNITIVE_BIAS**: Reciprocity bias - obligation feels natural rather than manipulation

**PERSONALITY_TRAIT**: Assess reaction to security policy enforcement requiring saying no to requests
**INSIDER_INDICATOR**: Difficulty with boundaries and relationship prioritization indicates higher risk
**COGNITIVE_BIAS**: Social proof bias - may reference others' approvals to justify exceptions

**PERSONALITY_TRAIT**: Evaluate knowledge documentation practices - does person hoard security information?
**INSIDER_INDICATOR**: Undocumented systems and processes indicate dependency creation
**COGNITIVE_BIAS**: Curse of knowledge - assumes others can't understand their security implementations

**PERSONALITY_TRAIT**: Observe response to new security team members or competing security expertise
**INSIDER_INDICATOR**: Territorial or undermining behavior indicates unhealthy Type 2 patterns
**COGNITIVE_BIAS**: Zero-sum bias - assumes others' security recognition reduces their value
