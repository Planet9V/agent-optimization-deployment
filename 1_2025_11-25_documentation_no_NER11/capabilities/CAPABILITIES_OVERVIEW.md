AEON's **7-level architecture** is a unique approach that separates data ingestion, knowledge construction, intelligence enhancement, and prediction into distinct layers—enabling **progressive intelligence enrichment** as data flows upward.

### Cross-Level Query Chains: From Equipment to Breach Prediction in One Query

**The Power of Multi-Level Queries**: AEON's architecture enables single Cypher queries that traverse all 7 levels, connecting equipment catalogs to breach predictions with complete business context.

#### Example 1: Complete Threat Analysis Query (Level 0 → Level 6)

**Business Question**: "Show me all equipment vulnerable to APT29 campaigns, calculate breach probability, and recommend actions with ROI"

**Single Cypher Query**:
```cypher

MATCH (level0:EquipmentCatalog {model: 'Cisco ASA 5525-X'})
  -[:INSTANTIATED_AS]->(level1:Equipment {organization: 'LADWP'})
  -[:HAS_VULNERABILITY]->(level2:CVE {id: 'CVE-2022-0778'})
  -[:EXPLOITED_BY]->(level3:ThreatActor {name: 'APT29'})
  -[:USES_TACTIC]->(level3b:AttackPattern {name: 'T1190'})
  -[:AFFECTED_BY_BIAS]->(level4:CognitiveBias {type: 'Normalcy Bias'})
  -[:ENHANCED_BY]->(level5:ThreatFeed {source: 'CISA KEV'})
  -[:PREDICTS]->(level6:BreachProbability)

// WHERE clauses for business logic
WHERE level1.criticality = 'HIGH'
  AND level2.cvss >= 7.0
  AND level3.sector_focus CONTAINS 'Water'
  AND level4.severity >= 8.0
  AND level5.active = true

// RETURN with complete business context
```

#### Example 5: Investment Prioritization (Level 2 → Level 4 → Level 6)

**Business Question**: "Of 316,000 CVEs, which should I patch first? Show me NOW/NEXT/NEVER priorities with business justification"

**Single Cypher Query**:
```cypher
// Level 2: All CVEs
MATCH (cve:CVE)

// Level 1: Affecting my equipment
MATCH (cve)<-[:HAS_VULNERABILITY]-(equipment:Equipment)
WHERE equipment.organization = 'LADWP'

// Level 3: Threat actor interest
OPTIONAL MATCH (cve)<-[:TARGETS]-(actor:ThreatActor)

// Level 4: Organizational context
MATCH (equipment)<-[:OWNS]-(org:Organization)
  -[:HAS_PSYCHOLOGY]->(psych:OrgPsychology)

// Level 6: Priority assessment (Enhancement 12)
MATCH (cve)-[:HAS_PRIORITY]->(priority:PriorityAssessment)
WHERE priority.organization = 'LADWP'

// Calculate combined score
WITH cve, equipment, actor, psych, impact, priority,
  (cve.cvss * cve.epss *
   CASE WHEN actor IS NOT NULL THEN 1.5 ELSE 1.0 END *
   equipment.criticality_weight *
   (365.0 / psych.patch_velocity_days)) AS priority_score

// Categorize into NOW/NEXT/NEVER
RETURN
  cve.id AS vulnerability,
  cve.cvss AS severity,
  COUNT(equipment) AS affected_equipment,
  actor.name AS active_threat_actor,
  impact.breach_cost AS potential_loss,
  impact.patch_cost AS remediation_cost,
  impact.roi AS investment_return,
  priority_score AS combined_score,

  CASE
    WHEN priority_score >= 8.0 THEN 'NOW'
    WHEN priority_score >= 5.0 THEN 'NEXT'
    ELSE 'NEVER'
  END AS action_category,

  CASE
    WHEN priority_score >= 8.0 THEN 'Emergency patching required within 14 days'
    WHEN priority_score >= 5.0 THEN 'Schedule in next maintenance window (30-90 days)'
    ELSE 'Monitor only, patch in annual cycle'
  END AS recommendation

ORDER BY priority_score DESC
LIMIT 200
```
